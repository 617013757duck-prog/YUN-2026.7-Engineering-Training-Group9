## 2. 接口协调分析

### 2.1 后端与AI服务的接口对接

#### 后端提供的数据结构
```
Visit（就诊记录）
├── patientId: Long
├── chiefComplaint: String
├── symptoms: List<Symptom>
└── status: VisitStatus
```

#### AI服务期望的请求格式
```json
{
  "visit_id": int,
  "symptoms": ["症状1", "症状2"],
  "patient_age": int,
  "duration_days": int,
  "additional_info": "string"
}
```

#### 协调分析

| 对接项 | 状态 | 说明 |
|--------|------|------|
| 症状数据 | ✅ 兼容 | 后端Symptom可转换为AI服务的symptoms数组 |
| 患者年龄 | ✅ 兼容 | 后端Visit.age可直接传递 |
| 风险等级 | ✅ 兼容 | AI服务返回risk_level，后端Visit.riskLevel可接收 |
| 分析结果 | ✅ 兼容 | AI服务返回analysis，后端Visit.aiAnalysis可接收 |

### 2.2 数据流设计

```
用户填写预问诊表单
    ↓
后端创建Visit记录（状态：PENDING）
    ↓
调用AI服务 /api/ai/analyze
    ↓
AI返回风险等级和分析结果
    ↓
后端更新Visit记录（状态：AI_ANALYZED）
    ↓
医生审核
    ↓
生成随访计划
```
